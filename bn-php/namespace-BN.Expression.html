Namespace BN\Expression

Classes summary
Associativity Determines how operators of the same precedence are grouped in the absence of brackets.
Brackets Implementation of brackets used in mathematical expressions. Left and right brackets are saved as two different strings.
ExpressionEvaluator For evaluation of mathematical expression in infix notation.
ExpressionPreparation Helper class for preparation of expression before parsing and evaluation. It adds spaces around brackets and operators. Spaces around minus operator is added only if minus is surrounded with numbers.
Operator Representation of mathematical operator with unique symbol, precedence, type of associativity and arguments count.
OperatorCallback Operator with defined @see IOperator and callback for evaluating. Used for calculation result of mathematical expression in postfix notation.
Operators Helper class for loading and creating operators for ExpressionEvaluator.
OperatorsFactory Factory for easier setup of all available operators for @see \BN\INumber numbers.
PostfixCalculator Implementation of algorithm for evaluating any postfix expression (RPN). Input queue can be taken from @see ShuntingYardParser.
ShuntingYardParser Parser for parsing mathematical expressions from infix to postfix notation (RPN). It's implementation of Dijkstra Shunting Yard algorithm. Result of parsing is queue with operands and operators (brackets are not needed in postfix notation)
Tokenizer Tokenize expressions by space. Array of tokens is available via iterator.
Interfaces summary
IBrackets ShuntingYardParser use brackets for grouping operands with operands, so default associativity and precedence can be changed with brackets. Object which implements this interface represents one type of brackets, e.g. round brackets where left bracket is ( and right is equal to ).
IOperator Operator in mathematical expression. Every operator should have string symbol, numeric precedence, type of associativity (left or right) and number of arguments. Used in ShuntingYardParser for parsing tokenized expression into postfix notation.
ITokenizer Used for breaking a text (in this case mathematical expression) into operators, operands and brackets. Parsed tokens are available only via iterator. Interface is also child of interface @see \IteratorAggregate. Used in @see ShuntingYardParser for tokenizing mathematical expressions.
Exceptions summary
MismatchedBracketException Exception thrown if there is mismatched left/right bracket in Shunting Yard algorithm.
NumberCallbackException Exception thrown if an argument in OperatorCallback is not instanceof INumber or if method in callback doesn't exists in INumber
ParserException Exception thrown if there error during Shunting Yard algorithm. It can be invalid value in expression or if tokenizer doesn't separate expression into strings.
PostfixException Exception thrown if there is error during evaluating of expression in postfix notation. Causes: - other value than operator (@see IOperator) or number (@see \BN\INumber) in expression - operator implements IOperator but callback is not definied in PostfixCalculator - if there too many or little arguments for operator (every operator has number of required arguments)
